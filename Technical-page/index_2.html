Two sides

This is a small section in content, but very big in importance. In order to accomplish web development, it is crucial to understand that the developer or developers are moving from a series of technical processes to a public environment that shows the web to the world. For this reason, there are some languajes that are procesed on the server side, whereas others are procesed on the browser side.

HTML5

HTML it's a tag languaje. The actual version is HTML5. This version is much more versatile than previous ones. In its beginning HTML was very simple. Every piece of information which appeared in the page was settled inside the body tag. There was, of course the head tag with all the technical information the page neded. And at the same time all was inside the html tag. But that was all.

HTML5 is the semantic version of HTML. It includes new tags which have a specific purpose. For instance:

  • The nav-bar in wich the menu with the links is settled uses the nav tag.
  • The purpose of the article tag is to divide the main secction of the page in well defined pieces of information.
  • Usually the pages have a seccion of related content wchich is placed using the aside tag.
  • At the bottom of the page the footer tag display our policies, our contact information or whatever we want to higlight so that our visitors can find what they are looking for.

CSS3

CSS is the web make up. The actual version is CSS3. The way in wich developers make use of this technology has changed through time.

In the early stages of web development, there was no special place to include CCS. Developers inserted small pieces of code inside the tag which was goin to be affected by its modification.

Later on, the code evolved and become more organized. The CSS was placed inside style tag, in the head of he document.

Nowadays, CCS3 has its own document which is linked from the HTML document.

But not just its placement has changed. With each version CSS has increased its power. Nothing to do with its first years in which CSS was used to change a font, a color or the formatting of a paragraph. Now it is possible to design images and even to create animations.

The only tool the developer had before in order to organise the content was the table element. But this has changed as well. An important characteristic of CSS is the ability it provides the developer to place each object on the page, with resources such as flex or grid.For instance the following snipet used in this page.

Media querries

Media querries are implemented inside CSS. The purpose of media querries is to create responsive pages, which means pages that adapt perfectly to the viewport, the device used in each case.

They allow to apply style in a different way to the same section regarding each device. So for instance, taking as an example this page, the script:


            @media (max-width:600px) {
            /*All this controls the mobile version*/  
           body{
               /*This makes the content to 
               be displayed in column*/ 
                display: flex;
                flex-flow: column;
            }
            nav li{
                /*This makes the navbar 
                to be displayed in two 
                rows to minimise space*/
               
                margin-top: 5px;
                display: inline-block;
               
            }
            #navbar{
                /*This makes the navbar 
                to be always visible*/
                position: sticky;
                top: 0px;
            }
            
        }
    


        @media (min-width:600px){
            /*With this I have a two columns 
            layout but just 
            for bigger devices*/
            #HTML5{
                columns:2;
                column-gap: 50px;
                column-rule: 3px solid blueviolet;  
            }
            .pareja{
                /*This controls the apearance 
                of this two pink blocks*/
                display: flex;
                flex-flow: row;
                align-items: center;
                justify-content: center;
                gap: 10px;
            }
        }

JavaScript

JavaScrip is, as everybody knows, what makes a website dynamic. It has a lot of potential. JavaScript supports functions, variables, constructors...

All of this makes the page react to the user actions. Any time the visitor press a button or enter a word in a form, the page shows a reaction as a consecuence. This is due to JavaScript.

Thanks to JavaScript it is posible to access to a specific item inside a matrix, or change a bit of text inside a string in a precise position.

PHP

All the languages seen so far are accesible from the browser. PHP is a server side language. It is used to manage delicate data such as forms, passwords and so on. In this way or websites are more secure.
